# Showcase

Regenerate the component showcase HTML from the current `system.md` tokens and component patterns.

---

## Phase 0: Load Design System

**Goal:** Read current state

**Actions:**

1. **Check for `docs/design/system.md`:**
   - If missing: STOP — "No design system found. Run `/chisel cut` first."

2. **Read `docs/design/system.md`:**
   - Parse all token values (colors, typography, spacing, shape, shadows, animation)
   - Parse component patterns section
   - Note the aesthetic direction

3. **Read `docs/design/tokens.css`:**
   - If missing or out of sync with system.md: regenerate it first

---

## Phase 1: Detect Components

**Goal:** Determine what to render in the showcase

**Actions:**

1. **From `system.md` component patterns section:** extract all listed components
2. **From codebase:** scan for component files that may have been added since last showcase
   - Glob for `src/components/**/*.{tsx,jsx,vue,svelte}`
   - Cross-reference with system.md list
   - If new components found: add them to the showcase, note in output

---

## Phase 2: Generate Showcase HTML

**Goal:** Build a self-contained showcase file

**Write `docs/design/showcase.html`:**

A single self-contained HTML file:

1. **Head:**
   - `<link rel="stylesheet" href="tokens.css">` (relative path)
   - Embedded styles for showcase layout only (grid, nav, sections)
   - Minimal JS for dark mode toggle and smooth-scroll nav

2. **Sidebar nav:**
   - Lists every component as a clickable link
   - Sticky, scrolls independently
   - Current section highlighted

3. **For each component, render a section containing:**

   **Header:** Component name

   **Variants grid:** All style variants side by side
   - Primary, secondary, outline, ghost, destructive (for buttons)
   - Default, error, disabled, focus (for inputs)
   - Adapt to what the component pattern defines

   **Size scale:** All sizes in a row (sm, md, lg)

   **States:** Show interactive states
   - Use real CSS `:hover`, `:active`, `:focus-visible`, `[disabled]`
   - For states that can't be shown simultaneously, label them

   **Dark mode:** The toggle switches the entire showcase

   **Token annotation:** Below each component, show which tokens it uses:
   ```
   Uses: --color-primary, --radius-md, --space-3, --shadow-sm
   ```

4. **Footer:** "Generated by Chisel · Last updated: YYYY-MM-DD"

**The showcase must be fully self-contained.** No build step, no imports beyond tokens.css. Pure HTML + CSS + minimal vanilla JS.

---

## Phase 3: Open and Report

**Actions:**

1. Open the showcase:
   ```bash
   open docs/design/showcase.html
   ```

2. Report what was generated:
   > "Showcase regenerated with N components. Open in your browser to review."

3. If new components were detected from codebase but not in system.md:
   > "Found N new components in codebase not in system.md: [list]. Added to showcase. Run `/chisel trim` to define their token patterns."
